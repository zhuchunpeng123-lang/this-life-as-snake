## A 级 · 架构级矛盾（护栏失效，但当前无错误行为）

### A1. CONFIG「深度冻结」从未生效 —— 双冻结互相抵消
- `02_config.js:5` 注释承诺：「深冻结由 core.js deepFreeze(CONFIG) 在启动自检执行」。
- 但 `02_config.js:319` 自己先做了顶层 `Object.freeze(CONFIG)`。
- `03_core.js:28`：`deepFreeze` 首行 `if (... || Object.isFrozen(obj)) { return obj }` —— CONFIG 顶层已冻结 → **立即早退，不递归子对象**。
- 结果：`CONFIG.PLAYER`、`CONFIG.COMBAT` 等**全部子对象从未被冻结**，任何运行时代码可静默篡改（`CONFIG.PLAYER.headRadius = 999` 不会报错）。AGENTS §一 宣称的「深度冻结」护栏实际是空的。
- 为什么至今没炸：`applyTuningOverrides()`（`03_core.js:194`）恰好靠"子对象未冻结"才能注入嵌套覆盖——**当前功能正常，纯护栏失效**。
- 连带风险：strict mode 下，若 tuning 覆盖里出现**单段 path**（无点，如 `'DEBUG'`），写顶层冻结对象会 `TypeError` 直接炸启动（`03_core.js` 全文件 `'use strict'`）。目前所有 path 均含点，未触发。
- `11_render.js:120` 注释「此刻已是…冻结值」同样基于错误假设（值是对的，"冻结"是假的）。
- 修复方向（二选一，需拍板）：① 删 `02_config.js:319`，让 core 的 `applyTuningOverrides→deepFreeze→selfCheck`（194-196 行，顺序本身正确）真正接管；② 保留现状，把两处注释改为「仅顶层冻结」并接受子对象可写。**推荐 ①**，一行删除，但属 02/03 联动，走 §八。

## B 级 · 确认的真 Bug（有错误行为）

### B1. GM「显示碰撞盒」蛇头圈静默不画 —— SPRITE_BASELINE 缺键
- `11_render.js:637`：`ctx.arc(s.head.x, s.head.y, getSpriteRadius('PLAYER.headRadius'), …)`。
- 但基线表 `11_render.js:121-124` 只有 `'PLAYER.headRadiusRender'` 和 `'PLAYER.bodyRadius'`，**没有 `'PLAYER.headRadius'`**。
- `getSpriteRadius`（126-129 行）：`base = undefined` → 若 GM 本次会话没拖过「蛇头判定半径」滑条（RT 无覆盖），返回 `undefined` → `arc(NaN)` 按 Canvas 规范**静默 no-op**（不抛错、不画）。
- 症状：开碰撞盒时**身体圈正常、蛇头圈缺失**；拖过滑条后头圈"凭空出现"。B-TUNE 标定的主目标恰是蛇头判定半径——标定工具本身在默认态是瞎的。
- 修复：基线表补 `'PLAYER.headRadius': PLAYER.headRadius` 一行（dev-only 改动，无 gameplay 影响）。

### B2. 结算面板玩家可见乱词「翁绞」+ 裸数字分母 5
- `12_ui.js:185`：`['发现的翁绞', '翁绞 ' + comboCount + ' / 5']` —— ①「翁绞」是乱码级错词（疑似"Combo/组合"的损坏），**面向玩家直接显示**；②分母 `5` 是裸数字（违 §六），且实装 Combo 只有 **3** 个（`CONFIG.COMBO` = steamExplosion / electroTurret / burningBarrage，`12_ui.js:7` COMBO_LABEL 同为 3 项）→ 玩家看到「3 / 5」永远集不满。
- GDD 口径：Combo 总数 5、MVP 交付 2-3 → 分母 5 是**终版口径**被提前写死进 MVP UI。
- 修复：文案改正 + 分母改 `Object.keys(CONFIG.COMBO).length` 动态取。纯 UI 改动。

## C 级 · 口径矛盾（gameplay 语义，修复会改手感，需拍板）

### C1. AOE 判定双口径：冰池「看到的=打到的」，火墙/护盾/蒸汽仍是 cell 级膨胀
- 冰池：`08_skill.js:124-126` 在 `enemiesIn` 候选集后做**精确圆-圆**二次过滤（注释自证「与渲染冰池圆一致」）。
- 火墙 `08_skill.js:88-89`、护盾球 `264-265`、蒸汽爆 `300-302`：`enemiesIn` 候选集**直接结算伤害，无精确过滤**。
- `enemiesIn`（48 行起）与 `SpatialHash.query` 同语义 = cell 级覆盖（`04_collision.js:30-36` 只算格子范围不算距离）；对照组：collision 主管线 query 后**都有** `circleHit` 精确判定（`04_collision.js:95、107`）。
- 后果：火/盾/蒸汽**实效半径系统性大于真源标称**（cellSize=64px，最坏膨胀超过一格对角）。真源写的 px 半径在这三处语义不成立。
- 对抗性自查：这不是回归——`08_skill.js:46` 注释自证 #6 优化「精确复刻 SpatialHash.query 返回集合、零行为变化」，即**历来如此**，现有手感就是在膨胀口径上调出来的。补精确过滤=实际削弱这三个技能。
- 拍板项：**方案 A** 维持现状，真源 §4 对应条目加注「cell 语义（实效≈标称+0~64px）」；**方案 B** 补精确过滤 + 按 §十 重调三处半径补偿。改哪个都要动真源，走 §七 顺序。

### C2. `RENDER.worldScale` 双源：config 值从不被读取
- `02_config.js` 的 `RENDER.worldScale: 0.8` 注释自称「仅文档真理源」；render 实际读 `RT('RENDER.worldScale', perfFB('worldScale', 0.8))` —— 优先级 = GM 覆盖 > 性能分档表（PERF tiers 各档 0.80/0.92/0.88/0.84）> 兜底 0.8。
- 即 **CONFIG.RENDER.worldScale 改了也没用**（MED 档设备实际跑 0.92）。这是已声明的分档设计，不算 bug，但「config 是数值真理源落地」的总口径在这一项上有豁口，容易误导后来调参者。建议 config 该行注释补一句「实际取值走 PERF.tiers，改这里无效」。

## W 级 · 脆弱点（当前自洽，一步之遥就出错）

### W1. `turnRateDecayPerSeg` 单位陷阱（×100 地雷）
- config `0.010`（小数/节）；GM 滑条单位「%/节」默认 `1.0`；热路径 `06_snake.js:25` = `RT('PLAYER.turnRateDecayPerSeg', P.turnRateDecayPerSeg * 100) / 100`。
- 当前自洽：该滑条在 TUNING_SCALAR 通道（只 rtSet 实时、不落 localStorage override、不回写 CONFIG）。
- 地雷：若日后把它挪进 override 通道（如 SNAKE_SCALAR 系），「保存并重载」会把 **1.0（百分比）** 写进 CONFIG（小数语义）→ 转向衰减放大 **100 倍**，蛇长几节就不会拐弯。同一 path 两种单位，靠"通道不同"隔离，无任何断言保护。

### W2. `aggroRangeByStage` 的 1-based RT path vs 0-based 数组
- `07_enemy.js:263-265`：读数组用 `stageId - 1`（0-based），RT path 却拼 `'…aggroRangeByStage.' + GS.stageId`（1-based 段号）。GM 滑条按段号写 `.2/.3/.4` → **当前配对正确**。
- 地雷：同一 path 字符串若进入 override 持久化通道，`setPath` 会按 **0-based 数组下标**写 CONFIG → 段位整体错一位（段②的值写进段③）。与 W1 同款「靠通道隔离的双口径」。
- 建议（W1+W2 一并）：editor 里给 TUNING_SCALAR 通道加注释红线「此列表内 path 永不迁入 override 通道；迁移前必须先统一单位/索引口径」。

### W3. 虚拟 RT path（config 中不存在的键）
- `09_wave.js` 读 `RT('PICKUP.gapEarly'/'PICKUP.gapFarm', …)`，config 里**没有**这两个键（真值是 `PICKUP.upgradeMinGapSecBySeg` 数组，与滑条默认 20/30 一致）。当前是"滑条↔wave 的私有桥名"，无 bug；但它们看起来像 CONFIG path，实际「保存并重载」对其无效（刷新即失）。易误导，建议命名前缀区分（如 `TUNE.gapEarly`）或注释标明。

### W4. `COMBAT.shake` 内嵌结构缩进误导
- `02_config.js:68-76`：`maxComposite / gateSec / steam` 确在 `shake` 对象内（render 读 `SHK.maxComposite` 路径一致 ✅），但缩进层级看起来像 shake 的平级。纯可读性，改缩进即可。
