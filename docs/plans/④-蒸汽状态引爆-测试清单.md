## 正常场景
- [ ] **火+冰 combo 激活 + 带冰敌入火墙 → 该敌位置引爆**
  复现：升 fire + ice（或 GM `debugActivateCombo('steamExplosion')`），让蛇身火墙扫过被冰减速/冻结的敌。
  预期：该敌位置出现 `fx:steamblast` 爆环（半径 90），爆心范围内的敌受 `fire.dotPerSec[fireLv]×2.5` 蒸汽伤害（沿用现有 steam 飘字）；**冰不单独造成伤害**（火墙 DOT 另算，属正常）。
- [ ] **同一带冰敌 2.0s 内只爆一次**
  复现：带冰敌持续停在火墙内观察数秒。
  预期：每约 2.0s 才在该敌位置爆一次（per-enemy 冷却 `COMBO_STEAM_INTERVAL_SEC`），非每帧狂爆。
- [ ] **离开火墙再进入、满 2.0s 后可再次引爆**
  复现：带冰敌离开火墙，>2.0s 后再回到火墙。
  预期：冷却已就绪 → 重新引爆一次。
- [ ] **多带冰敌同帧在火墙内 → 各自独立引爆**
  复现：多个带冰敌同时处于火墙不同位置。
  预期：同帧出现多个 `fx:steamblast`（各在各自敌位），冷却互不影响。
- [ ] **全程无闪电/雷；电磁 combo 表现不受影响**
  复现：同时观察 electroTurret/lightning（若有）。
  预期：④ 不触发任何闪电；电磁连锁仍走 `fx:electroarc` 紫、B-4 表现不变。

## 边界 / 反向场景
- [ ] **反向：只升 fire（无 ice）→ 无蒸汽**
  复现：仅升 fire，不升 ice。
  预期：`foundCombo.steamExplosion` 不激活，**无任何 `fx:steamblast`**（火墙 DOT 仍正常）。
- [ ] **边界：带冰敌不在火墙内（仅在冰区、未触火）→ 不引爆**
  复现：敌被冰减速但蛇身火墙未扫到它。
  预期：不引爆（必须火扫到；验证「火焰光环扫到」前置）。
- [ ] **边界：带冰敌死亡 → 冷却无残留**
  复现：带冰敌被蒸汽（或他因）击杀后，新刷敌/训练假人（`debugSetSkill`+`spawnDummy`）进场。
  预期：新生敌 `steamCd=0`（对象池复位），不会带上一任的冷却残留。
- [ ] **边界：per-enemy 冷却独立性**
  复现：A 敌引爆瞬间，B 敌也满冷却且带冰在火墙。
  预期：B 同帧引爆（验证冷却是 per-enemy，非全局共享）。
- [ ] **边界：Lv5 冻结敌也能被引爆**
  复现：ice 满级（Lv5 冻结 `pct=1, slowT=lv5FreezeSec`），火墙扫过冻结敌。
  预期：冻结敌 `slowT>0` 满足带冰判定 → 正常引爆（验证无独立 `frozenT` 字段也能覆盖冻结态）。
