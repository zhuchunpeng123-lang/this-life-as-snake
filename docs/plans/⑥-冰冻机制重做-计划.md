# ⑥ 冰冻机制重做：常驻 trail → CD 自动索敌冰池

> 来源：Notion AI 建议 + 用户确认（先完成后6，再 2/5）。真源 §4.2/§9 已回写，数值按标注从仓取值。
> 流程：本计划由 窗口A 落地代码 → 用户浏览器实测 → 回「⑥绿」→ 窗口A 分件 commit + tag baseline-b9。

## 1. 目标一句话
冰冻不再是蛇尾走过滞留的减速带，改为**每 freezeCd 秒自动触发一次**，复用飞镖/闪电索敌逻辑锁定射程内最密敌群，在落点生成一个存续 poolLingerSec 的冰池；池内敌人吃减速→冻结；带火蛇身扫过池内被冻敌仍触发蒸汽爆炸（④ 一行不动）。落点全自动、无手动施法、不绑头/尾（视觉仅从尾部甩出）。

## 3. 具体改动点
- **config**：`freezeCd=3.0`、`poolLingerSec=4.0`、`poolRadius=[20,26,32,40,50]`、`seekRange=[100,140,180,220,260]`、`slowPct=[0.20,0.30,0.40,0.50,0.60]`（已定值）、`lv5FreezeSec=1.0`。均初值，留 ③ 标定。
- **索敌**：以蛇头为源（`headPos()`，与飞镖/闪电一致），`seekRange` 内候选敌群中，取「以该敌为中心、半径 `poolR` 邻域内敌数最多」者为落点；射程内无敌人则不落池。
- **落池**：对象池化冰池 `{x,y,r,life,expire}`，存续 `poolLingerSec`；`Bus.emit('fx:ice_pool',{from:尾点,to:落点,r,life})`。
- **减速施加**：每帧对冰池做圆-圆判定（含敌半径，与渲染圆严格一致）→ `En.applySlow(e,pct,slowWin)`；`pct=slowPct[i]`，Lv5 `pct=1`、`slowWin=lv5FreezeSec`，否则 `slowWin=poolLingerSec`。进入检测 + `fx:iceslow` 飘字沿用。
- **RT 桥**：`freezeCd/poolLingerSec/poolRadius[i]/seekRange[i]/slowPct[i]` 全部走 `RT()` 接 editor 运行时覆盖（③ 标定即时生效）。

- `07_enemy.js`：未动（debuff 施加与蒸汽引爆逻辑不变）。

## 5. 验收标准
- ✅ 正常：持有 ice 后，约每 3s 自动在敌群最密处落一个冰池；池内敌人出现蓝环 `drawSlowMark` 减速，Lv5 冻结（pct=1）明显停住。
- ✅ 联动：火墙扫过池内被冻敌 → 蒸汽爆炸照常触发（`fx:steamblast`）。
- ✅ 边界①（无目标）：射程内无敌人时，不落池、不报错、不崩。
- ✅ 边界②（重开）：`core:run_reset` 清空冰池 + 复位 `timer.ice`，重开无残留/NaN。
- ✅ 边界③（无致死源）：冰池只减速/冻结，不产生任何伤害（伤害仍仅来自火/闪电/护盾/蒸汽）。
- ✅ 表现：冰锥从尾部飞向落点（`fx:ice_pool`），冰池随寿命淡出。
