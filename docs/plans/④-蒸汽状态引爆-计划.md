# ④ 蒸汽状态引爆 · §四 执行计划

> 设计已锁定（Notion 计划页 ④「锁定设计」）。本计划按 AGENTS.md §四 模板产出，供用户核 + 回写 §9 后 Craft 落地。
> 落地前置：触发口径重定位属 gameplay 触发定义变更，**等用户在 Notion §9 回写「蒸汽 combo 触发口径」条目后**再进 Craft（用户明确：回写 §9 后落地）。

## 1. 目标一句话
将蒸汽 combo（火+冰）由「蛇头每 2s 周期 AOE」改为「火焰光环扫到带冰敌人时，在该敌位置引爆蒸汽 AOE」，按 per-enemy 2.0s 冷却节流；引爆源=火焰、不引入闪电/雷，冰只作触发条件（无直伤）。

## 2. 涉及文件及理由
- `snake55/08_skill.js`（主改动）：`tickCombos` 蒸汽分支改为「火墙内 + 带冰 + 冷却就绪 → 该敌位置引爆」；移除全局 `timer.steam` 周期触发。
- `snake55/07_enemy.js`（状态读取 + 冷却字段）：`newEnemy` 加 `steamCd` 字段；`spawn`/`spawnBullet`/`spawnDummy` 复位 `steamCd=0`；`updateOne` 每帧 `steamCd -= dt`（死亡经对象池复用自然复位，满足「死亡清理」）。
- 复用（不改）：`05_particle.js`/`11_render.js` 的 `fx:steamblast`（爆环视效，已支持多实例）；`02_config.js` 的 `CO.steamExplosion.{radius,damageMul}`、`SK.fire.{dotPerSec,radius,segStep}`、`COMBO_STEAM_INTERVAL_SEC`（已冻结 2.0）。
- **无新数值**（用户口径 + §十）：全程复用既有常量，不增 `cooldownSec`/`damageMul`/任何裸数字。

## 3. 具体改动点（分条）

### 3.1 ⑦ enemy.js：新增 per-enemy 蒸汽冷却字段
- `newEnemy()` 默认字段加 `steamCd: 0`（与 `slowT`/`burnT` 同区）。
- `spawn()`（L68 复位块）加 `e.steamCd = 0`；`spawnBullet()`（L88）、`spawnDummy()`（L100）同理。
- `updateOne(e, dt, sm)`：在 `slowT` 衰减附近（L235）加 `if (e.steamCd > 0) { e.steamCd -= dt }`。
- 死亡清理：敌人死亡 → `pool.release` → 下次 `spawn`/`spawnBullet`/`spawnDummy` 复位 `steamCd=0`，无残留（满足需求，无需在 `die()` 额外处理）。

### 3.2 ⑧ skill.js：`tickCombos` 蒸汽分支重写
当前（L181-191）：
```js
if (foundCombo.steamExplosion) {                 // 火+冰：周期 AOE
    timer.steam -= dt
    if (timer.steam <= 0) {
        timer.steam = COMBO_STEAM_INTERVAL_SEC
        var h = headPos(), es = enemiesIn(h.x, h.y, CO.steamExplosion.radius)
        for (var k = 0; k < es.length; k++) {
            hurtCombo(es[k], SK.fire.dotPerSec[idx('fire')] * CO.steamExplosion.damageMul, false, 'steam')
        }
        Bus.emit('fx:steamblast', { x: h.x, y: h.y, radius: CO.steamExplosion.radius })
    }
}
```
改为：
```js
if (foundCombo.steamExplosion) {                 // 火+冰：火墙扫到带冰敌 → 该敌位置引爆（④）
    var fi = idx('fire')
    var fr = RT('SKILL.fire.radius.' + fi, SK.fire.radius[fi])   // 火墙半径，与 tickFire 一致
    var segs = segmentsList(), step = SK.fire.segStep[fi] || 1
    var inFire = {}                                               // 本帧处于火墙内的敌人 id 集合
    for (var s = 0; s < segs.length; s += step) {
        var fs = enemiesIn(segs[s].x, segs[s].y, fr)
        for (var k = 0; k < fs.length; k++) { inFire[fs[k].id] = true }
    }
    var all = allEnemies()
    for (var n = 0; n < all.length; n++) {
        var e = all[n]
        if (!e.slowT || e.slowT <= 0) { continue }   // 带冰判定 = 复用冰减速/冻结 debuff（slowT>0）
        if (!inFire[e.id]) { continue }              // 火焰光环扫到
        if (e.steamCd > 0) { continue }              // per-enemy 节流（复用 COMBO_STEAM_INTERVAL_SEC=2.0，不新增数值）
        e.steamCd = COMBO_STEAM_INTERVAL_SEC          // 重置该敌冷却
        var es2 = enemiesIn(e.x, e.y, CO.steamExplosion.radius)
        for (var m = 0; m < es2.length; m++) {
            hurtCombo(es2[m], SK.fire.dotPerSec[fi] * CO.steamExplosion.damageMul, false, 'steam')  // 引爆源=火焰，冰无直伤
        }
        Bus.emit('fx:steamblast', { x: e.x, y: e.y, radius: CO.steamExplosion.radius })   // 爆环对准真实爆心（该敌位置）
    }
}
```
- 移除 `timer.steam -= dt` / `timer.steam <= 0` 全局周期触发块。
- `run_reset`（L280）中 `timer.steam = 0` 一并清理（或保留无害；建议移除）。

### 3.3 设计决定（需你确认）
- **「带冰状态」判定 = `e.slowT > 0`**（冰减速/冻结 debuff 的 lingering 窗口），**非** `inIce`（仅冰区在场标记）。理由：你明确「复用冰减速/冻结 debuff（lingerSec>0）」，lingerSec 即 slow 跟随窗口；`inIce` 只是进入检测标记，不代表残留 debuff。
- **引爆 AOE 伤害范围内所有敌人**（`enemiesIn(e.x, e.y, radius)`，含触发敌自身及附近非冰敌）—— 沿用当前 AOE 语义，仅爆心从蛇头移到带冰敌位置。
- 每个带冰敌独立 2.0s 冷却 → 多敌可同帧各自引爆（多个 `fx:steamblast`）。
- 闪电/雷完全不牵入（④ 明确砍掉）；`tickLightning`/`doLightningChain`/`electroTurret` 不动。

## 5. 验收标准
**正常场景**
- [ ] 火+冰 combo 激活，带冰敌进入火墙范围 → 该敌位置出现 `fx:steamblast` 爆环，范围敌受 `fire.dotPerSec[fireLv]×2.5` 蒸汽伤害（沿用现有 steam 飘字标签）；冰不单独造成伤害。
- [ ] 同一带冰敌 2.0s 内只引爆一次；离开火墙再进入、满 2.0s 后可再次引爆。
- [ ] 多个带冰敌同帧在火墙内 → 各自独立引爆（多个爆环），冷却互不干扰。
- [ ] 全程无闪电/雷出现；电磁 combo 表现不受影响。

**边界 / 反向场景**
- [ ] 反向：只升 fire（无 ice）→ steam combo 不激活，无任何 `fx:steamblast`（火墙 DOT 仍正常）。
- [ ] 边界：带冰敌**不在**火墙内（仅在冰区、未触火）→ 不引爆（必须火扫到）。
- [ ] 边界：带冰敌死亡（被蒸汽或他因击杀）→ `steamCd` 随对象池复位，新生敌/假人无残留冷却。
- [ ] 边界：per-enemy 冷却与全局无关 —— A 敌引爆瞬间，B 敌若也满冷却且带冰在火墙 → 同帧引爆（验证独立性）。
