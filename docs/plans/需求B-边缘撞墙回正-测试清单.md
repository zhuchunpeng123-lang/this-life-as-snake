# 需求 B · 边缘/墙角/撞墙回正 — 浏览器实测清单

> 配合 `需求B-边缘撞墙回正.md`。强刷 `Ctrl+Shift+R` 清缓存后实测（build 见 commit）。

## 复现步骤与预期

### 1. 贴单边墙滑行（正常）
- 步骤：进游戏 → 摇杆贴着**上墙**走一段 → 再换下/左/右墙各走一段。
- 预期：蛇沿墙**顺滑滑行**、不转圈、不失控；蛇头不再死顶墙外（目视头朝已贴合墙切向）。

### 2. 离开墙 / 反向拨杆回正（正常 + 反向）
- 步骤：贴墙滑行时，把摇杆拨向**墙内/反向**。
- 预期：朝向**平滑回正**、操控即时恢复；中心区手感与未撞墙时一致（无回归）。

### 3. 四个墙角（边界）
- 步骤：分别把蛇头从四个方向**顶进墙角**（两轴同撞）。
- 预期：不横跳、不抖、不原地绕；墙角处蛇头角度自由（不被乱拉）。

### 4. 墙角双轴冻结后拨出（验收加测 2）
- 步骤：蛇头挤进墙角（两轴同撞）→ 保持一会 → 把摇杆拨向「离开墙角」的方向。
- 预期：**立即顺滑滑离**墙角、不横跳、不卡死。

### 5. 正对墙反向 180°（验收加测 1）
- 步骤：蛇头**正对墙**前进 → 玩家猛拨**反向**（约 180°）。
- 预期：平滑 180° 转回、不卡死、不抖；贴墙期间无「失控打转」。

### 6. 贴墙拨离跟手（验收加测 3 · 补充 a）
- 步骤：贴墙滑行中，把摇杆从「贴墙」快速拨到「离墙方向」。
- 预期：朝向**立刻跟手回正**、操控即时恢复；无「被墙吸住/对拉」的迟滞感（验证收敛在玩家拨墙外时跳过）。

### 7. 与 A 摇杆联动无回归（验收加测 4）
- 步骤：全程用摇杆操纵，刻意多撞几次墙。
- 预期：撞墙收敛自然、不抢摇杆输入；**松手直行**遇墙仍沿墙滑（不丢输入、不瞬移）。

### 8. 硬卡 diff（验收加测 5）
- 步骤：B commit 后执行 `git show --stat <B_commit>` 或 `git diff <A>..<B>`。
- 预期：`03_core.js` / `04_collision.js` / `14_main.js` / `docs/《数值真理源 §9》` **必须为空**；仅 `06_snake.js` + 本计划/STATUS/测试清单变动。

## 反向/异常确认
- 不撞墙时：转向/恒速/手感与改前完全一致（无回归）。
- 宽容窗：撞墙后 0.2s 内拨离仍顺滑（grace 复用，非新增救回窗）。
- 刮擦减速仍在：贴墙期速度明显低于开放区（验证补充 b，wallScrapeSpeedMult 未废）。

## 桌面 / 手机双端
- 桌面：鼠标**按住拖动**=摇杆；**鼠标 hover 不再自动转向**（已屏蔽，仅摇杆+键盘）。
- 手机：touch = 摇杆（右下角固定锚点），同上 1–8 项一致。
